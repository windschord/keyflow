## 背景

### 問題の概要

再生中にメトロノームのチェックをOFF→ONすると、それ以降クリックが一切鳴らなくなる。
停止→再再生でも復帰せず、回復経路は楽譜の再読み込みのみ。

（分析レポート: `docs/sdd/troubleshooting/2026-07-07-metronome-feedback/analysis.md`）

### 根本原因（実測で確認済み）

- 一度 `stop()` した `Tone.Sequence` を `start(0)` で再開しても、内部 Part に停止イベントが
  残るため二度と発火しない（tone@15.1.22）。
- `metronome.ts:17` の `if (!this.sequence)` ガードがシーケンスをキャッシュしており、
  再有効化時は死んだシーケンスへ `start(0)` を呼ぶだけになっている。
- `setEnabled(true)` で毎回シーケンスを再生成する修正案を一時適用した実測で、
  OFF→ON後もクリックが継続することを確認済み。

### 関連する仕様

- REQ-006-005: メトロノーム有効時、設定されたテンポでクリック音を鳴らす
- 承認済み追加調整: 「1拍目強調」OFF時の通常拍 velocity を 0.6 から 1.0 へ変更する
  （アクセントOFF時は強弱の差が不要なため。2026-07-07承認）

## 実装内容

### 修正対象

- ファイル: `src/renderer/src/lib/audio-engine/metronome.ts`
  - `setEnabled(true)` で既存シーケンスがあれば `dispose()` して `null` に戻し、
    毎回新しい `Tone.Sequence` を生成して `start(0)` する
    （`if (!this.sequence)` キャッシュの廃止）。
  - クリックコールバックの velocity 判定を変更する: アクセント有効時は
    一拍目 `'C6'`・1.0 / 他拍 `'C5'`・0.6（現状どおり）、アクセント無効時は
    全拍 `'C5'`・1.0 とする。
  - `rebuildSequence()` は `setEnabled(true)` が毎回再生成するようになるため、
    実装を `setEnabled` の再呼び出しに単純化できる（重複コードを残さないこと）。
- ファイル: `src/renderer/src/lib/audio-engine/audio-engine.test.ts`
  - 下記テスト項目を追加・更新する。

TDDで進める。
