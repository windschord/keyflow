# DEC-003: 運指エンジンをTypeScriptで独自実装（Python不採用）

## ステータス: 承認済

## 決定内容

運指提案エンジンをPython/music21ではなく、TypeScriptで独自実装する。
実行はRendererプロセスのWeb Worker上で行う。

## 背景と検討

| 選択肢 | メリット | デメリット |
|--------|---------|-----------|
| **TypeScript独自実装（採用）** | Pythonランタイム不要、Electronにバンドル可、低遅延IPC | アルゴリズムの実装コストが高い |
| Python + music21（不採用） | 実績ある高品質アルゴリズム、実装コスト低 | Pythonインストールが必要（NFR-C-008違反） |
| Python バンドル（不採用） | Pythonランタイムをインストーラーに同梱 | インストーラーが +200MB以上になる、macOSアーキテクチャ対応が複雑 |
| WASM化されたC/C++ライブラリ | 高性能 | 適切なライブラリが現状存在しない |

## 決定理由

1. **要件 NFR-C-008〜010**: ユーザーへの追加ランタイムインストール不要
2. Parncutt-Terzuoloモデルは数式として公開されており、TypeScriptで再実装可能
3. Web WorkerはUIスレッドと分離されるため、計算中も60fpsのUI描画を維持できる
4. 将来的にONNX Runtime JSへ置き換える際もインターフェースを変えずに移行可能

## 実装参考文献

- Parncutt, R., Sloboda, J. A., Clarke, E. F., Raekallio, M., & Desain, P. (1997). An Ergonomic Model of Keyboard Fingering for Melodic Fragments. *Music Perception*, 14(4), 341–382.
- Balliauw, M., et al. (2015). Generating fingering for piano pieces. *ISMIR 2015*.
