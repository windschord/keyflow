## 背景

### 問題の概要

TASK-056で鍵盤数プリセット（88/76/61/49鍵）を導入した結果、キャンバス幅がウィンドウ幅より狭い場合に鍵盤が左寄せで表示される。左右の余白がページ背景の白のままであり、鍵盤との境界に違和感がある（2026-07-06 PR#26実機フィードバック）。

### 関連する仕様

- US-005（画面鍵盤ガイド）
- `src/renderer/src/components/PianoKeyboard/index.tsx`（キャンバスのコンテナ）

## 実装内容

### 修正対象

- ファイル: `src/renderer/src/components/PianoKeyboard/index.tsx`

### 実装手順

1. キャンバスをコンテナ内で水平センタリングする。キャンバス幅がコンテナより広い場合（88鍵×狭いウィンドウ等）は従来どおり横スクロールでき、先頭が切れないこと（CSSの `justify-content: safe center` 相当の挙動）
2. コンテナの背景色をヘッダーバーと同じ `#e0e0e0`（`App.tsx` のヘッダー背景と同値）にする
3. テスト: コンテナのスタイル（センタリング・背景色）と、狭い鍵盤プリセット選択時にもクリック座標→MIDI変換が正しいこと

### 注意事項

- クリック座標の計算は `canvas.getBoundingClientRect()` 基準のため、センタリングによるオフセットの影響を受けないことをテストで担保する
- 背景色の `#e0e0e0` はヘッダーバー（`App.tsx`）と同値にする。値を変える場合は両方を揃えること
