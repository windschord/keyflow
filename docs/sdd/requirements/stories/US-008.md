# US-008: 運指メモの書き込み

## ユーザーストーリー

**私は** ピアノ練習者として
**〜したい** 楽譜上に自分の運指（指番号）や練習メモを書き込みたい
**なぜなら** 自分なりの運指や気づきを楽譜に直接記録して後で参照したいから

## 受入基準（EARS記法）

### 機能要件

- **REQ-008-001**: ユーザーが音符を右クリックした時、システムは運指番号（1〜5）を入力できるメニューまたはポップアップを表示しなければならない
- **REQ-008-002**: 運指番号が設定された時、システムはその音符の上に指番号を楽譜上に表示しなければならない
- **REQ-008-003**: ユーザーが小節や音符にテキストコメントを追加できるよう、システムはノートコメント機能を提供しなければならない
- **REQ-008-004**: システムは書き込まれた運指とメモをMusicXMLファイルとは別のアノテーションファイル（.json）として保存しなければならない
- **REQ-008-005**: ユーザーが同じMusicXMLファイルを再度開いた時、システムは保存済みのアノテーションを自動的に復元しなければならない
- **REQ-008-006**: ユーザーが運指番号を削除したい時、システムは右クリックメニューから削除できなければならない

## 画面/UI要件

- 運指番号は楽譜上の音符の上部に小さな数字（赤色）で表示する
- 編集モードと閲覧モードを切り替えるトグルボタンを設ける（誤操作防止）
- コメントはツールチップまたは吹き出しで表示する

## データ要件

アノテーションファイル形式（例）:
```json
{
  "musicXmlPath": "path/to/file.xml",
  "annotations": [
    {
      "measureNumber": 1,
      "noteIndex": 0,
      "fingerNumber": 3,
      "comment": "親指から始める"
    }
  ]
}
```
