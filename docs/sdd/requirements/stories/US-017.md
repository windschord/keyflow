# US-017: 楽譜ライブラリ（一覧管理画面）

## ユーザーストーリー

**私は** 複数の楽譜で練習するアプリ利用者として
**〜したい** 取り込んだ楽譜をアプリ内のライブラリ画面で一覧・検索し、そこからすぐ開きたい
**なぜなら** 毎回ファイルダイアログで楽譜を探し直すのは手間であり、練習する曲の管理と
再開をアプリ内で完結させたいから

## 受入基準（EARS記法）

### 機能要件

- **REQ-017-001**: ユーザーが楽譜ファイルを開いた時（ダイアログ・ドラッグ＆ドロップ・
  ライブラリのいずれ経由でも）、システムはその楽譜をライブラリへ自動登録しなければならない。
  登録内容はファイルパス・タイトル・作曲者（いずれもMusicXMLから抽出、欠落時は
  ファイル名で代替）・登録日時・最終利用日時とする
- **REQ-017-002**: 既にライブラリに存在するファイルパスの楽譜が開かれた時、システムは
  重複登録せず、既存エントリの最終利用日時とメタデータを更新しなければならない
- **REQ-017-003**: ユーザーがライブラリ画面を開いた時、システムは登録済み楽譜の一覧を
  タイトル・作曲者・最終利用日時とともに表示しなければならない
- **REQ-017-004**: ユーザーが検索欄に文字列を入力した時、システムはタイトルまたは作曲者に
  部分一致（大文字小文字を区別しない）するエントリのみを表示しなければならない
- **REQ-017-005**: ユーザーが並べ替え条件を選択した時、システムはタイトル・登録日時・
  最終利用日時のいずれかで昇順・降順に一覧を並べ替えなければならない
- **REQ-017-006**: ユーザーがエントリを削除した時、システムは確認を求めたうえで
  ライブラリから削除しなければならない。楽譜ファイル本体と運指メモ
  （.annotation.json）は削除してはならない
- **REQ-017-007**: ユーザーが一覧のエントリを選択した時、システムはその楽譜を既存の
  読み込みフロー（パースエラー処理・読み取り許可リスト登録を含む）で開かなければならない
- **REQ-017-008**: エントリのファイルがディスク上に存在しない場合、システムは一覧上で
  欠損を示す表示をし、開く操作時にはエラーを通知して当該エントリを削除する選択肢を
  提示しなければならない
- **REQ-017-009**: システムはライブラリをアプリデータフォルダへ永続化し、アプリ再起動後も
  復元しなければならない
- **REQ-017-010**: アプリ起動時に楽譜が読み込まれていない場合、システムはライブラリ画面を
  初期表示しなければならない（ライブラリが空の場合は空状態の案内とファイルを開く導線を表示）
- **REQ-017-011**: ライブラリ画面のUI文言は多言語対応（US-016のi18n機構）に
  従わなければならない
- **REQ-017-012**: 楽譜を開いた状態でライブラリ画面を表示した場合、システムは楽譜を
  選び直すことなく元の楽譜表示へ戻れる導線を提供しなければならない（2026-07-13の
  実機フィードバック起点。楽譜未読み込み時は戻る導線を表示しない）

### 対象外（本ストーリーのスコープ外）

- タグ付け・お気に入り・フォルダ分類
- 楽譜サムネイル（譜面プレビュー画像）の生成
- 練習統計（正解率履歴等）の蓄積・表示（練習履歴の永続化は未実装のまま）
- 楽譜ファイル本体のアプリ管理領域へのコピー（ファイルは元の場所を参照し続ける）
- フォルダ監視による自動取り込み

## 画面/UI要件

- ライブラリ画面はヘッダーの専用ボタンからいつでも開ける（楽譜表示との切り替え）
- 一覧の各行にタイトル・作曲者・最終利用日時を表示し、行クリックで開く
- 検索欄・並べ替えセレクタを一覧上部に配置する
- 削除は各行の操作（ボタンまたはメニュー）から実行し、確認ダイアログを挟む
- 欠損ファイルの行は視覚的に区別する（グレーアウト+欠損ラベル等）

## データ要件

- ライブラリは設定（AppSettings）とは別のストアとして永続化する
- エントリ: `{ path, title, composer, addedAt, lastOpenedAt }`（pathが一意キー）
- 読み込み時はスキーマ・値域を検証し、不正な要素は除外するフェイルソフト方式
  （アノテーション読み込みのTASK-092と同方針）
- 既存の「最近使ったファイル」（recentFiles、最大10件）は当面併存させる。
  SettingsModal内の表示は維持し、将来の統合は別ストーリーで判断する

- セキュリティ: ライブラリから開く場合も、Mainプロセス側で拡張子検証と読み取り許可リスト
  （PathAllowlist）への登録を必ず経由する（TASK-086の防御を迂回しない）
- タイトル・作曲者の抽出はRendererのパース結果を利用する（Mainで再パースしない）
