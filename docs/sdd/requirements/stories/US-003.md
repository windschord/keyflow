# US-003: 右手/左手分離練習

## ユーザーストーリー

**私は** ピアノ練習者として
**〜したい** 右手専用・左手専用・両手の練習モードを切り替えて練習したい
**なぜなら** 難しい箇所を手ごとに分けて集中的に練習したいから

## 受入基準（EARS記法）

### 機能要件

- **REQ-003-001**: ユーザーが練習モードを「右手のみ」に設定した時、システムは右手パートのMIDI入力のみを正誤判定しなければならない
- **REQ-003-002**: ユーザーが練習モードを「左手のみ」に設定した時、システムは左手パートのMIDI入力のみを正誤判定しなければならない
- **REQ-003-003**: ユーザーが練習モードを「両手」に設定した時、システムは全パートのMIDI入力を正誤判定しなければならない
- **REQ-003-004**（保留・将来拡張）: 練習モードが「右手のみ」または「左手のみ」の間、システムはアプリ内音源で非練習パートを自動伴奏しなければならない
  - 2026-07-05のスコープ変更: 曲の再生はUS-010（お手本演奏）を正とし、練習中の非練習パート自動伴奏は将来拡張へ降格する。理由: 正誤判定待ち（waitモード）の練習進行は時間軸で進まないため、時間軸で進む自動伴奏と同期させるには追加の設計（判定位置に追従する伴奏スケジューラ）が必要となるためである。加えて、現行の練習体験はUS-010の再生で代替できるためである（docs/sdd/troubleshooting/2026-07-05-test-escape/analysis.md M7）。なお、2026-07-05のTASK-051（REQ-010-010）でUS-010の再生自体も現在の練習モードで絞り込まれるよう変更されたため、片手練習中に「もう一方の手の音」を聴きたい場合は練習モードを「両手」へ切り替えたうえで再生する必要がある
- **REQ-003-005**: ユーザーが練習モードを切り替えた時、システムは現在の再生位置を維持したまま即座に切り替えを完了しなければならない

## 画面/UI要件

- 画面上部またはツールバーに「右手」「左手」「両手」の切替ボタンを配置する
- 現在のモードが一目でわかるよう選択中のボタンをハイライト表示する

## データ要件

- MusicXMLのパートとMIDIチャンネルの対応を設定として保持する
- 練習モードの設定は練習セッション内で保持する

## 備考

- MusicXMLでパートが1つしかない場合は「両手」モードのみ有効にする
- 将来的に「特定パートのみ」選択も拡張可能な設計にする
- 曲全体を聴きたい場合は練習モードを「両手」に切り替えたうえで [US-010](US-010.md) の再生機能を使用する。2026-07-05 TASK-051（REQ-010-010）により、再生は現在の練習モードで絞り込まれる＝片手練習中の再生はその手のみが鳴るよう変更したため、全パートを聴くには「両手」モードへの切替が必要。片手練習中の非練習パート自動伴奏（REQ-003-004）は将来拡張として保留中
