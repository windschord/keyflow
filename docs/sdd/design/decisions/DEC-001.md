# DEC-001: Electronを採用（TauriやFlutter不採用）

## ステータス: 承認済

## 決定内容

フロントエンドフレームワークとしてElectron + React/TypeScriptを採用する。

## 背景と検討

| 選択肢 | メリット | デメリット |
|--------|---------|-----------|
| **Electron** | OSMDがWeb技術ベースで即採用可、Web MIDI API使用可、最成熟エコシステム | インストーラーサイズが大きい（~100MB） |
| Tauri | Electronより軽量、Rustバックエンドで高性能 | MIDIライブラリが未成熟、OSMDとの統合事例が少ない |
| Flutter | クロスプラットフォームUI | MIDIサポート未成熟、楽譜レンダリングライブラリがない |
| Python + Qt | music21が直接使えた | **Pythonランタイム依存**があり要件(NFR-C-008〜010)に反する |

## 決定理由

1. OpenSheetMusicDisplay (OSMD) がWebベースで、Electronと相性が最もよい
2. MusicXML対応・MIDI・Web Audio APIが全てWeb技術で完結する
3. 要件「ユーザーへのランタイムインストール不要」をElectronのバンドル機構で満たせる
4. Windows/macOS両対応を同一コードベースで実現できる

## 影響

- インストーラーサイズは ~80〜120MB 程度（Chromiumを含むため）
- 関連: [NFR-C-008〜010](../../requirements/nfr/compatibility.md)
