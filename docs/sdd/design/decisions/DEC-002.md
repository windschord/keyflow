# DEC-002: 楽譜レンダリングにOSMDを採用

## ステータス: 承認済

## 決定内容

MusicXMLの楽譜レンダリングに **OpenSheetMusicDisplay (OSMD)** を採用する。

## 背景と検討

| 選択肢 | 説明 |
|--------|------|
| **OSMD** | MusicXML直接入力対応、高品質レンダリング、活発なOSS |
| VexFlow | 低レベルAPIで柔軟だがMusicXMLパーサーは別途必要 |
| LilyPond | 最高品質だがサーバーサイド処理が必要、デスクトップ統合が複雑 |
| Verovio (WASM) | 高品質、WASMで動作するが日本語コミュニティが小さい |

## 決定理由

- OSMDはMusicXML 3.1を直接サポートし、独自パーサーが不要（レンダリング目的では）
- MIT License で商用利用可能
- Electronとの統合実績が多い
- カーソル・ハイライトAPIが組み込みで提供されている

## 注意点

- 正誤判定・運指計算用には別途 **MusicXML Parser** コンポーネントで内部データモデルを構築する（OSMDの内部表現は変更される可能性があるため）
