# US-016: UIの多言語対応（日本語・英語切り替え）

## ユーザーストーリー

**私は** 日本語を読めない、または英語UIを好むアプリ利用者として
**〜したい** アプリのUI表示言語を日本語と英語から選びたい
**なぜなら** UI文言が日本語のみだと日本語話者以外がアプリを使えず、OSSとして国際的に配布する妨げになるから

## 受入基準（EARS記法）

### 機能要件

- **REQ-016-001**: システムはUI表示言語として日本語と英語の2言語をサポートしなければならない
- **REQ-016-002**: 言語設定が未保存の初回起動時、システムはOSロケールが日本語（`ja`系）の場合は日本語を、それ以外の場合は英語をUI表示言語として選択しなければならない
- **REQ-016-003**: ユーザーが設定画面で言語を切り替えた時、システムはアプリ再起動なしにRendererのUI文言を即時切り替え、選択した言語を設定として永続化しなければならない
- **REQ-016-004**: ユーザーが言語を切り替えた時、システムはアプリケーションメニュー（メニューバー）の文言も同じ言語へ再構築しなければならない
- **REQ-016-005**: アプリ起動時、システムは永続化された言語設定をRenderer・Mainプロセス双方のUI文言に適用しなければならない
- **REQ-016-006**: 表示中の言語に対応する翻訳文言が欠落している場合、システムは英語の文言をフォールバックとして表示しなければならない
- **REQ-016-007**: 言語切り替えは、ヘッダー・設定モーダル・Aboutモーダル・コンテキストメニュー・統計表示・エラーダイアログ/トースト通知を含む全UI文言に適用されなければならない

### 対象外（本ストーリーのスコープ外）

- 楽譜内の文言（曲名・歌詞等、MusicXMLファイル由来のデータ）の翻訳
- ライブラリ名・音源クレジット等の固有名詞の翻訳
- README等リポジトリドキュメントの多言語化（別途対応済み）
- 日本語・英語以外の言語の追加（翻訳リソースの追加のみで対応可能な構造とする）

## 画面/UI要件

- 言語切り替えは設定モーダル（SettingsModal）のUIセクションに配置する
- 言語の選択肢はそれぞれの言語の自己表記で表示する（「日本語」「English」）。この2つの選択肢自体は言語切り替えの影響を受けない
- 切り替え時に画面のリロードやちらつきを発生させない

## データ要件

- `AppSettings.ui.language: 'ja' | 'en'` として electron-store に永続化する
- キー欠落時（既存ユーザーの設定ファイル）は未設定として扱い、REQ-016-002のOSロケール判定を適用する（DEFAULT_SETTINGSとの浅いマージによる後方互換を維持）

## 備考

- 現状のUI文言はコンポーネント内に日本語がハードコードされているため、翻訳キーへの置き換え（externalization）が実装の主作業になる
- Mainプロセス（メニューバー）とRendererで翻訳リソースを共有できる構造が望ましい（二重管理の防止）
