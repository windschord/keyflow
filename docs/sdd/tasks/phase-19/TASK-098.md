## 実装内容

### 言語セレクタUI（REQ-016-003）

- `components/SettingsModal/index.tsx` のUIセクションに言語セレクタを追加する
- 選択肢は「日本語」「English」の自己表記固定（言語切り替えの影響を受けない）
- 変更時: ui-sliceの `setLanguage` を呼び（即時反映）、`settings:set` で
  `ui.language` を永続化する（既存の設定保存パターンに従う）
- 保存値が `'auto'` の場合、セレクタは現在の解決結果（ja/en）を表示してよいが、
  ユーザーが明示選択するまで `'auto'` を上書きしない実装が望ましい。実装が複雑になる
  場合は「明示選択時に'ja'/'en'を保存する」動作を最低ラインとする

### 文言外部化

- `components/SettingsModal/`（セクション見出し・各設定項目ラベル・選択肢）
- `components/AboutPanel/`（AboutModal含む。見出し・説明文。ライブラリ名・
  ライセンス種別名・Salamanderクレジット等の固有名詞は対象外）
- `App.tsx` ほかのエラーダイアログ・トースト文言（パース失敗・ファイル読み込み失敗等）
- MIDIデバイス関連の表示文言（SettingsModal内のデバイス選択等）

### 注意事項

- 音色表示名（グランドピアノ等）がvoices.ts側に定義されている場合、表示層で翻訳キーへ
  マップする（ドメイン定義のIDは変更しない）
- AboutPanelのクレジット文言はCC-BY表記義務に関わるため原文を維持する

## 完了サマリー

言語セレクタはSettingsModalの独立セクションに追加した。
セレクタの選択肢は自己表記固定（「日本語」「English」）であり、表示言語に影響されない。

変更時はui-sliceの`setLanguage`を即時反映し、`settings:set('ui', ...)`で永続化する。
保存失敗時は他設定項目と同じパターンでロールバックする。

保存値が`auto`の間はセレクタに現在の解決結果（ja/en）のみを表示する。
ユーザーが明示選択するまで`auto`自体は上書きしない（起動時resolveLanguageの結果を尊重）。

SettingsModal本体・AboutPanel・AboutModal・App.tsxのエラーダイアログとドロップヒントを外部化した。
音色表示名（グランドピアノ等）はドメインIDを変えずvoiceNames名前空間へのマッピングで対応した。
AboutPanelのSalamanderクレジット・ライブラリ名・ライセンス種別名は原文のまま維持した。

追加した翻訳キーの名前空間は`settings`（既存を拡張）・`about`・`app`・`voiceNames`の4件。
