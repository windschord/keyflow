# US-018: Windowsの配布形態拡充（arm64対応・インストール不要版）

## ユーザーストーリー

**私は** さまざまなWindows端末を使うアプリ利用者として
**〜したい** Windows on ARM端末でも動くビルドと、インストールせずにすぐ使えるビルドを選びたい
**なぜなら** Surface Pro X等のARM64端末ではx64インストーラーが最適に動かないから。
また管理者権限がなかったり試用目的だったりする環境では、インストール作業なしでアプリを起動したいから

## 受入基準（EARS記法）

### 機能要件

- **REQ-018-001**: システムのWindows向けリリース成果物は、x64とarm64の2アーキテクチャを提供
  しなければならない
- **REQ-018-002**: システムはWindows向けに、従来のNSISインストーラーに加えて、インストール
  不要で起動できるportable版（単一実行ファイル）を提供しなければならない
- **REQ-018-003**: portable版は、実行時にインストール作業（レジストリ変更・スタートメニュー
  登録・管理者権限）を要求せずに起動できなければならない
- **REQ-018-004**: 各Windows成果物のファイル名は、配布形態（インストーラー / portable）と
  アーキテクチャ（x64 / arm64）が識別できる命名でなければならない
- **REQ-018-005**: 追加したWindows成果物は、既存のリリースワークフローと同様に、ビルド来歴
  証明（attestation）とSHA256チェックサムの付与対象に含まれなければならない

### 対象外（本ストーリーのスコープ外）

- Windows 32bit（ia32）向けビルド（現行の32bit専用Windowsは実質絶滅のため対象外）
- macOSの配布形態変更（現行のdmg/zip×x64/arm64を維持）
- コード署名・公証（引き続き未対応。初回起動時のOS警告は既知の制約）
- 自動アップデート（NFR-C-007。現時点で未実装のまま）

## データ要件

- 該当なし（配布・ビルド設定の変更のみ）

- NFR-C-006（配布形態）: Windowsの配布形態にportable版を追加
- NFR-C-008/009（自己完結）: portable版もElectronのバンドルNode.jsを含み、外部ランタイム
  不要で全機能が動作すること

## 備考

- ネイティブアドオン非依存（MIDIはRendererのWeb MIDI API）のため、arm64 Windowsビルドは
  クロスビルドで生成可能（`npmRebuild: false`）
- 実際のarm64/portableビルドの成立確認はローカルmacOS環境では限定的である。
  ネイティブなNSIS/portable生成にwineを要するためである。
  リリースワークフロー（windows-latestランナー）での通し検証を正とする
  （プレリリースタグでの検証、TASK-085/093と同運用）
