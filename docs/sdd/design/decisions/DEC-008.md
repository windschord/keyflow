# DEC-008: ライブラリライセンス一覧をビルド時に自動生成

## ステータス

承認済（2026-07-07）

## コンテキスト

US-015でアプリ内ライセンス表示（Aboutページ）が要件化された。ライブラリ一覧を手動メンテナンスすると、依存の追加・更新時に漏れ・陳腐化が必ず発生する（REQ-015-004）。

## 検討した選択肢

| 選択肢 | 評価 |
|--------|------|
| A) 自前スクリプトでnode_modules走査（`scripts/generate-licenses.mjs`） | 採用。本番依存が少数（十数件）のため自前走査で十分。devDependencies追加なし |
| B) `license-checker` 系ライブラリ | 高機能だがdevDependency追加とメンテ先が増える。依存規模に対して過剰 |
| C) 手動メンテナンスのMarkdown/JSON | 漏れ・陳腐化が不可避で不採用（REQ-015-004違反） |

## 決定

- `scripts/generate-licenses.mjs`（自前実装）で `dependencies` を走査し、`src/renderer/src/generated/licenses.json` を生成する
- `predev` / `prebuild` npmフックで自動実行し、生成物はgitignoreする（常に実依存と同期）
- バージョン表示は electron-vite の `define` で `package.json` の `version` を `__APP_VERSION__` としてビルド時注入する（表示とpackage.jsonの一致を機械的に保証、IPC追加なし）
- 自動収集できない音源クレジット（Salamander）は静的データ（`credits.ts`）として併記する

## 影響

- 依存を追加するだけでAboutページの一覧に自動反映される
- 生成前にRendererをビルドすると `licenses.json` が無くビルドエラーになるため、npmフックの結線が必須（タスクで検証する）
