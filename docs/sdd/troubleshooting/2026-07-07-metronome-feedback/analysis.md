# 分析レポート: メトロノーム実機フィードバック3件（2026-07-07）

## 1. 問題事象

PR #27（Phase 14）の実機確認で、ユーザーから以下3件のフィードバックがあった。

| # | 報告内容 | 分類 |
|---|---------|------|
| 1 | メトロノームはチェックを入れただけで（再生していなくても）音が鳴ってほしい | 仕様変更（REQ-006-009 新設） |
| 2 | 再生中はテンポ切り替えができないので、設定を変えられないようにしてほしい | 仕様変更（REQ-006-010 新設） |
| 3 | 「1拍目強調」のチェックを外したら、それ以降メトロノームの音が鳴らなくなった | バグ |

## 2. 調査方法

Playwright for Electron でビルド済みバイナリを起動し、`metronome.ts` のクリック
コールバックに一時的な計測ログ（クリックごとの発火時刻・アクセント判定・例外）を
仕込んで実測した。調査スクリプトはスクラッチパッドで実行し、計測コードは
調査後に revert 済み（コミットに含まれない）。

## 3. 実測結果

| 操作 | 結果 |
|------|------|
| 再生中（アクセントON）3秒 | クリック6回（アクセント2/通常4）。正常 |
| 「1拍目強調」をOFF | クリックは継続（3秒で7回、例外なし）。**OFF単独では止まらない** |
| アクセント再ON | 正常に復帰 |
| **再生中にメトロノームをOFF→ON** | **以降クリック0回（無音）** |
| 無音化後に停止→再再生 | クリック0回（**復帰しない**） |
| テンポスライダー100%→200%（再生中） | クリック平均間隔 501ms→248ms（**正確に追随**） |

## 4. 根本原因（事象3）

一度 `stop()` した `Tone.Sequence` を `start(0)` で再開しても、内部 Part に
停止イベントが残るため二度と発火しない（tone@15.1.22。上記の実測で確認）。

`metronome.ts` の `setEnabled` は `if (!this.sequence)` ガードでシーケンスを
キャッシュしており、再有効化時は既存シーケンスに `start(0)` を呼ぶだけである。
このため「再生中にメトロノームをOFF→ON」した時点でクリックが永久に停止し、
停止→再再生でも復帰しない（回復経路は楽譜の再読み込みのみ）。

ユーザー報告は「1拍目強調を外したら鳴らなくなった」というものだったが、実測すると
アクセントOFF単独ではクリックは継続する。アクセントOFF後の通常拍は velocity 0.6 と
控えめなため音が変わったと感じ、メトロノームのチェックを入れ直した結果、
本バグに到達したものと推定する。

### 修正の実証

`setEnabled(true)` で既存シーケンスを dispose して毎回再生成する修正案を
一時適用して再計測したところ、OFF→ON後もクリックが継続し（3秒で12回@240bpm）、
停止→再再生も正常であることを確認した。

## 5. 事象2の補足: テンポ変更は再生中も有効である

実測すると、スライダー変更は再生中も正確に追随した（間隔501ms→248ms）。
数値入力欄は Enter またはフォーカス移動で確定する実装のため、「効かない」という
体感は入力未確定に起因した可能性がある。ただしユーザー要望は「再生中は
変更できないようにする」であり、REQ-006-010 としてテンポ設定UIのロックを実装する。

## 6. 修正方針（承認済み 2026-07-07）

### TASK-065 [BugFix] メトロノーム再有効化で無音（Sequence毎回再生成）

- `setEnabled(true)` で既存シーケンスを dispose して毎回再生成する
  （`if (!this.sequence)` キャッシュを廃止。`rebuildSequence()` と同じ要領）。
- あわせて「1拍目強調」OFF時の通常拍 velocity を 0.6 から 1.0（従来の音量）へ変更する
  （承認済み。アクセントOFF時は強弱の差を付ける必要がないため）。
  アクセントON時は従来どおり一拍目 C6・1.0 / 他拍 C5・0.6 とする。

### TASK-066 メトロノーム単独再生（REQ-006-009）

- メトロノーム有効中、Transport が動いていない間は `Tone.Clock` による独立クロックで
  クリックを鳴らす（チェックONだけで即座に鳴り始める）。
- 拍子は `Score.timeSignature.beats`（未ロード時は4）を使い、クリック開始を1拍目として
  カウントし、1拍目強調オプションに従ってアクセントを付ける。
- テンポはテンポスライダーの現在値（`Transport.bpm` と同じ値）に追随する。
- 再生開始で独立クロックを止めて楽譜同期のシーケンスへ切り替え、
  一時停止・停止で独立クロックへ戻る。

### TASK-067 再生中のテンポ設定UIロック（REQ-006-010）

- `playbackState === 'playing'` の間、テンポスライダー・数値入力・リセットボタンを
  無効化する。一時停止・停止中は変更可能とする。
