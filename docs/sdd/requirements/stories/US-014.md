# US-014: ペダル記号の再生反映

## ユーザーストーリー

**私は** ピアノ練習者として
**〜したい** 楽譜に書かれたペダル記号がお手本演奏の響きに反映されてほしい
**なぜなら** ペダルの有無で曲の印象が大きく変わり、実際の演奏イメージを掴むための手本にならないから

## 受入基準（EARS記法）

### 機能要件

- **REQ-014-001**: MusicXMLの解析時、システムは `<direction>` 内の `<pedal type="start|stop|change">` を解析し、ペダル区間（開始tick〜終了tick）としてScoreデータに保持しなければならない
- **REQ-014-002**: ペダル区間内に発音されるノートの再生時、システムはそのノートのリリースを記譜上の音価ではなくペダル区間の終了時点（`stop` または次の `change`）まで延長しなければならない
- **REQ-014-003**: `<pedal type="change">` を検出した時、システムはその時点で保持中の響きを解放し、同時点から新しいペダル区間を開始しなければならない（踏み替え）
- **REQ-014-004**: もし楽曲にペダル記号が存在しないならば、システムは従来どおり記譜上の音価で再生しなければならない（既存楽曲の再生結果を変えない）
- **REQ-014-005**: 再生停止またはループ折り返しの時、システムは延長中の全ノートを即座に解放しなければならない（音の残留・重複発音の防止）

## データ要件

- `Score` 型にペダル区間の配列を追加（例: `pedalSpans: Array<{ startTick: number; endTick: number }>`）。`change` は区間の分割として表現する
- 同一ノートがペダルにより延長される場合も、正誤判定・カーソル表示・鍵盤ガイドは記譜上の音価を基準とする（本ストーリーは再生音のみに影響）

## 備考

- 対象はダンパーペダル（サステイン）のみ。ソステヌート・ソフトペダル、`<pedal type="continue">`（ペダル線の継続表現）の音響反映はスコープ外
- 同一音高がペダル保持中に再打鍵された場合は、直前の同音を解放してから発音する（Tone.Samplerの同音重複による音量肥大を防ぐ）
- ペダル終了が明示されないまま曲が終わる場合は、曲末尾を区間終了として扱う
