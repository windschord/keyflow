# US-011: アプリのブランディング（アイコン・ウィンドウタイトル）

## ユーザーストーリー

**私は** ピアノ練習者として
**〜したい** アプリのアイコンとウィンドウタイトルがこのアプリ固有のものであってほしい
**なぜなら** Electronデフォルトのままではタスクバー・Dockで他アプリと区別できず、完成度の低い印象を受けるから

## 受入基準（EARS記法）

### 機能要件

- **REQ-011-001**: アプリ起動時、システムはウィンドウタイトルに「MusicXML Piano Practice」を表示しなければならない（`index.html` の `<title>` および BrowserWindow の `title` の両方で設定する）
- **REQ-011-002**: アプリ実行中、システムはタスクバー・Dock・ウィンドウにピアノ+楽譜モチーフの独自アイコンを表示しなければならない
- **REQ-011-003**: パッケージビルド時、システムは同一デザインのアイコンを全対象OS向け形式（macOS: `build/icon.icns`、Windows: `build/icon.ico`、Linux/実行時: `resources/icon.png`）で含まなければならない。これには現在欠落している `build/icon.ico` の解消を含む

## 画面/UI要件

- アイコンのマスターはSVGとしてリポジトリに保管し、各形式（png/icns/ico）を再生成可能にする
- アイコンは小サイズ（16px）でも判別可能なシンプルな図案とする

## 関連ストーリー

- なし（独立した改善）

## 備考

- `electron-builder.yml` の `productName` は既に「MusicXML Piano Practice」であり、ウィンドウタイトルもこれに合わせる
- 開発モード（`npm run dev`）ではmacOSのDockアイコンはElectronデフォルトのままでも許容する（パッケージ版で独自アイコンが表示されることを合格条件とする）
