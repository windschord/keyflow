## 背景

### 問題の概要

「曲全体の再生（お手本演奏）」がユーザーストーリー（US-001〜009）に存在せず、ユーザーが曲を開いても再生して聴く手段がない。フェーズAで再生ボタン・Spaceキーの暫定結線（TASK-026）を行ったが、対応する機能要件が存在しないため、受入基準・仕様の裏付けがない状態が続いている。

### 根本原因

- 要件定義時にSynthesiaライク体験の核である「お手本演奏を聴く」ユースケースがスコープから漏れた（`docs/sdd/troubleshooting/2026-07-04-app-unusable/analysis.md` 原因2）。
- `docs/sdd/requirements/nfr/usability.md:33` に「スペース: 再生/停止」というショートカット記述があるにもかかわらず、対応する機能要件（ユーザーストーリー）が存在せず、NFRと機能要件が矛盾している。
- `AudioEngineService`（`src/renderer/src/lib/audio-engine/index.ts`）には `loadAccompaniment` / `playAccompaniment` / `setBpm` / `setMetronomeEnabled` が実装済みだが、要件不在のため統合タスク（TASK-016）のスコープに結線が含まれなかった。

### 関連する仕様

- NFR-U-003: キーボードショートカット「スペース: 再生/停止」（`docs/sdd/requirements/nfr/usability.md:30-36`）— 対応する機能要件が不在
- REQ-003-004 / US-006: 非練習パート自動伴奏・テンポ/メトロノーム — エンジン実装済みだがUI結線ゼロだった（フェーズAで暫定対応）
- `docs/sdd/requirements/index.md`: ユーザーストーリー一覧・機能要件サマリ（US-010 / REQ-010 の追記対象）

## 実装内容

### 修正対象

- ファイル: `docs/sdd/requirements/stories/US-010.md`（新規作成）
- ファイル: `docs/sdd/requirements/index.md`（ユーザーストーリー一覧・機能要件サマリ・ドキュメント構成ツリーへのUS-010/REQ-010追記）
- 変更内容: sdd-documentationの `requirements-defining` スキルを用いて、US-010「曲の再生（お手本演奏）」をEARS記法で定義する

### 実装手順

1. `sdd-documentation:requirements-defining` スキルを起動し、既存ストーリー（`docs/sdd/requirements/stories/US-004.md` 等）のフォーマットを確認する
2. `docs/sdd/requirements/stories/US-010.md` を新規作成し、EARS記法で以下の受入基準を定義する
   - 再生: ユーザーが再生操作（再生ボタンまたはスペースキー）を行ったとき、システムは読み込み済みの楽曲を現在の再生位置から演奏しなければならない
   - 一時停止: 再生中にユーザーが一時停止操作を行ったとき、システムは再生位置を保持したまま演奏を停止しなければならない
   - 停止: ユーザーが停止操作を行ったとき、システムは演奏を停止し再生位置を先頭（またはループ開始位置）に戻さなければならない
   - カーソル連動: 再生中、システムは楽譜上のカーソルを再生位置に同期して移動させなければならない
   - テンポ反映: ユーザーがテンポを変更したとき、システムは再生速度へ即時（または次の再生開始時に）反映しなければならない
3. NFR-U-003の「スペース: 再生/停止」記述との整合を確認し、US-010の受入基準からNFR-U-003を相互参照する（必要に応じて `nfr/usability.md` に補足を追記する）
4. `docs/sdd/requirements/index.md` を更新する
   - ユーザーストーリー一覧に US-010（優先度: 高）を追加
   - 機能要件サマリに REQ-010「楽曲全体の再生・一時停止・停止（お手本演奏）」を追加
   - ドキュメント構成ツリーに `stories/US-010.md` を追加

### 注意事項

- EARS記法（WHEN/WHILE/WHERE/IF-THEN + SHALL）を既存ストーリーと同じ粒度で用いること
- 「Synthesiaスタイルの落下ノートビジュアル」はスコープ外のまま維持する（`docs/sdd/requirements/index.md:73`）。US-010はあくまで「音による再生＋カーソル連動」であり、落下ノーツUIを要求しない
- 再生とMIDI練習（US-004の正誤判定）が同時に動作する場合の振る舞い（再生中はMIDI判定を停止するか等）を要件として明記すること
- 一時停止と停止を区別して定義すること（分析レポートの期待動作「曲を開いたら再生して聴ける」を満たす最小セット）
- ループ範囲（US-007）設定時の再生挙動（ループ内リピート）との関係を備考に記載すること
