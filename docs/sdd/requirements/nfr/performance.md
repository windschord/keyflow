# 性能要件

## 概要

MusicXMLピアノ練習アプリの性能・応答性に関する非機能要件を定義する。
ピアノ演奏はリアルタイム性が極めて重要であり、特にMIDI処理のレイテンシは
人間が知覚できる閾値（約20ms）以下に抑える必要がある。

## 要件一覧

| ID | 要件概要 | 目標値 | 優先度 |
|----|---------|--------|--------|
| NFR-P-001 | MIDI入力レイテンシ | < 10ms | 必須 |
| NFR-P-002 | 楽譜レンダリング時間 | < 3秒（A4 10ページ） | 必須 |
| NFR-P-003 | MusicXMLインポート時間 | < 5秒（標準曲） | 必須 |
| NFR-P-004 | テンポ変更の反映 | < 50ms | 必須 |
| NFR-P-005 | 運指提案計算時間 | < 30秒（1楽章分） | 推奨 |
| NFR-P-006 | アプリ起動時間 | < 5秒（コールドスタート） | 推奨 |
| NFR-P-007 | メモリ使用量 | < 500MB（通常動作時） | 推奨 |

## 詳細要件

### NFR-P-001: MIDI入力レイテンシ

- **NFR-P-001**: システムはMIDIキーボードからのノートオンイベントを受信してから10ms以内に音符認識処理を完了しなければならない
- 根拠: 人間が知覚できる音楽的なレイテンシの閾値は約20ms。10msは十分なバッファを持つ値。

### NFR-P-002: 楽譜レンダリング

- **NFR-P-002**: MusicXMLファイルを読み込んだ時、システムはA4サイズ10ページ相当の楽譜を3秒以内にレンダリングしなければならない
- 根拠: ユーザーの待機許容時間の一般的な基準値

### NFR-P-003: MusicXMLインポート

- **NFR-P-003**: 一般的なピアノ曲（1〜2パート、100小節程度）のMusicXMLファイルを5秒以内に読み込んで解析しなければならない

### NFR-P-005: 運指提案

- **NFR-P-005**: Pythonエンジンによる運指提案は、一楽章分（約100小節）の計算を30秒以内に完了しなければならない
- 根拠: バックグラウンド処理なので30秒まで許容できるが、それ以上はユーザー体験を損なう

## 測定方法

- MIDI レイテンシ: 専用テストツールで測定（Web MIDI Benchmark等）
- レンダリング時間: パフォーマンスプロファイラーで計測
- 運指計算時間: Python側でtimeモジュールで計測
