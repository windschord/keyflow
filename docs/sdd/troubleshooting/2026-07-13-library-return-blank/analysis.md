# 分析レポート: 「楽譜へ戻る」後に楽譜表示まで数秒間真っ白になる

## 問題事象

| 項目 | 内容 |
|-----|------|
| 現象 | ライブラリ画面から「楽譜へ戻る」を押すと、楽譜が表示されるまで数秒間真っ白になる |
| 期待動作 | REQ-017-012の設計どおり、表示切り替えのみで即座に元の楽譜が表示される |
| 再現手順 | 楽譜（実曲規模）を開く→ライブラリ画面を表示→「楽譜へ戻る」を押す |
| 発生環境 | feat/score-library（TASK-105実装後）、`npm run dev` 実機確認 |
| 報告 | 2026-07-13 ユーザー実機フィードバック |

## 根本原因

TASK-103の「常時マウント+`display:none`で隠す」方式と、TASK-049で導入した
ResizeObserverによる自前再描画制御の相互作用。

1. `src/renderer/src/components/ScoreRenderer/osmd-controller.ts` はOSMDの
   `autoResize` を無効化し、ResizeObserverの発火から250msデバウンスで
   `handleResize()`（`osmd.render()`+noteIdマップ再構築+オーバーレイ再適用）を実行する
2. ライブラリ表示時、`App.tsx` が楽譜コンテナを `display:none` にするとコンテナの
   サイズが0×0になり、ResizeObserverが発火する。デバウンス後に**幅0のまま
   `osmd.render()` が実行され、表示済みのSVGが破棄**される（真っ白の直接原因）
3. 「楽譜へ戻る」でサイズが復元すると再度ResizeObserverが発火し、デバウンス後に
   **楽譜全体の同期再レンダリング**が走る。実曲規模では数秒を要し、完了までコンテナは
   空のままとなる

補足: E2E（TASK-105）は小さなサンプル楽譜のため再レンダリングが一瞬で完了し、
本事象を検出できなかった。往復後の状態保持（テンポ値）は検証していたが、
「表示までの時間」は検証対象になっていなかった。

## 仕様との照合

- REQ-017-012（US-017）は「楽譜を選び直すことなく元の楽譜表示へ戻れる導線」を要求し、
  設計（score-library.md）は「`setActiveView('score')` を呼ぶだけで楽譜の再読み込みは
  発生しない」と定めている
- 実装はMusicXMLの再読み込み（再パース）こそしないものの、**OSMDの再レンダリングが
  発生しており設計意図（即時復帰）と乖離**している
- 分類: **実装バグ**（仕様は正しく、隠し方（display:none）が再描画機構と整合していない）

## 修正方針の候補

### 案A: 不可視・同一サイズ時の再レンダリング抑止（根本対策、推奨）

`osmd-controller.ts` の `handleResize()` に以下のガードを追加する。

- コンテナが不可視（`clientWidth === 0 || clientHeight === 0`）の間は
  再レンダリングせずスキップする（SVGを破棄しない）
- 直近の描画時サイズを記録し、**サイズが変わっていなければ再レンダリングしない**
  （隠す→同一サイズで戻すの往復が完全に無再描画・即時になる）
- 不可視中に `setZoom` 等で描画要求が発生した場合は保留フラグを立て、
  可視復帰時のResizeObserver発火で一度だけ再レンダリングする

利点: 復帰が即時になり、隠している間の無駄な0幅renderも消える。変更は
osmd-controller内に閉じており、ウィンドウリサイズ時の既存挙動（TASK-049）は不変。
リスク: 低（ガード条件の単体テストで担保可能）

### 案B: 隠し方の変更（display:none → visibility:hidden + absolute配置）

サイズを維持したまま隠せばResizeObserverが発火しない。ただしLibraryViewとの
重ね合わせレイアウト管理が必要になり、App.tsxのレイアウト変更が広範囲に及ぶ。
リスク: 中（レイアウト回帰・既存結線テストへの影響）

### 案C: 再レンダリング中のスピナー表示（対症療法）

数秒の空白をスピナーで置き換える。無駄な再レンダリング自体は残る。
案Aで即時復帰が実現できるため、単独では推奨しない。

## 推奨

**案A**。設計意図（即時復帰）を直接実現し、変更範囲が最小。案Aで復帰は即時になる
ためスピナーは不要になる見込み。将来的に長い描画の残る経路を発見した場合は
案Cを追加検討する。

## 影響範囲

- `src/renderer/src/components/ScoreRenderer/osmd-controller.ts`（ガード追加）
- 単体テスト追加（0サイズスキップ・同一サイズスキップ・保留フラグ）
- E2E: 戻り即時性の検証（戻った直後にSVGが存在することのassert強化）を検討
