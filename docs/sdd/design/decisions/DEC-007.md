# DEC-007: ヘッダーを1行+ポップオーバー方式へ再設計

## ステータス

承認済（2026-07-07、ユーザー選択）

## コンテキスト

現行UIはApp.tsx上段バー + Toolbarの2ブロックが実質3〜4行を占有していた。(a) 譜面表示領域が狭い、(b) ボタンが多く直感的でない、(c) ヘッダーと設定モーダルの内容分類が不明瞭という課題があった（US-012）。

## 検討した選択肢

| 選択肢 | 評価 |
|--------|------|
| A) 1行集約 + 低頻度操作をポップオーバーへ | 採用。譜面領域最大化（ヘッダー約48px=現状の約1/4）。Synthesiaライクな見た目 |
| B) 2行にグループ整理 | 全操作常時可視だが圧縮率は約1/2どまり |
| C) 現構成のまま高さのみ圧縮 | 「ごちゃごちゃ感」「分類の不明瞭さ」が解消されず不採用 |

## 決定

- ヘッダーは1行（高さ48px、最大56px）とし、以下の分類基準で操作を配置する:
  - **ヘッダー（常時表示）** = 練習中によく触るもの: ファイルを開く・再生/一時停止/停止・ループ・テンポ/BPM・練習対象
  - **ポップオーバー（QuickPanel）** = ときどき調整するもの: 音量・表示倍率・運指表示/提案・メトロノーム・正解率
  - **設定モーダル** = 一度決めたら変えないもの: 既定値・MIDIデバイス・鍵盤表示・音色・About
- 既存の子コンポーネント（PracticeModeSelector等）はロジックを変えず再配置し、機能の喪失を禁止する（REQ-012-004）
- デスクトップ（マウス主体）前提で、クリック領域の最小を44px→36pxに緩和する（nfr/usability.mdの44px基準はタッチ前提のため）

## 影響

- `Toolbar/index.tsx` とApp.tsx上段バーは廃止され、`Header/` に一本化される
- design/components/toolbar.md は本決定により [header.md](../components/header.md) に置き換えられる
- E2Eテストのセレクタ（ボタン配置）に追随修正が必要
